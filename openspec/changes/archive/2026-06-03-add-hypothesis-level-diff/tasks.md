## 2. `compute_diff` 追加 hypothesis 集合差（D-5，继承防污染门）

- [x] 2.1 在 `compute_diff` 现有 finding 集合差**之后、`return` 之前**追加 hypothesis 段：键 `hyp_key(h) = frozenset(h.supporting_findings)`；建键前排除 `supporting_findings` 为空的假设（两侧）并把条数累加进 `hypothesis_unanchored`（两 run 合计、可重复计数）；按键集合差产 `hypothesis_added`（current 键 ∉ baseline 键集）/ `hypothesis_resolved`（反之）；键同时存在两侧且 `confidence` 不同 → `hypothesis_confidence_changed`。**两处排序分别明确**：①建键映射前对每报告 hypotheses 按 `(sorted(supporting_findings), confidence, description)` 稳定排序以固定碰撞场景的确定性 representative；②三列表发射前按键的可读形式 `sorted(supporting_findings)` 排序（确定性输出）。投影模型的 `supporting_findings` 字段须 `sorted(set(...))`（去重+排序，渲染/审计稳定；键本身由 frozenset 派生、天然去重）。**禁止** `description` 入键。验收：单测——不同 supporting_findings 集 → added/resolved；同集异 confidence → confidence_changed；同集同 confidence 异 description → 不进任何列表（证只看证据集不看 description）；`["A","A"]` 与 `["A"]` 键相同（去重）→ 匹配为同一假设；**current 含 2 条同键（同 confidence 异 description）新键假设 → `hypothesis_added` 恰好 1 条 representative（per-report 排序首条），非 2 条**；同输入两次调用三列表逐元素等且顺序一致（确定性）。
- [x] 2.2 退化键处理（D-2/D-3）：空 supporting_findings 假设计入 `hypothesis_unanchored` 且不进任何列表；一报告内同键多条（碰撞）时 added/resolved 仍按键集合正确（碰撞安全），confidence 比较仅对两侧均「该键恰 1 条」的键发射、歧义键（任一侧 >1 条）确定性跳过**并把该键计入 `hypothesis_ambiguous_keys`**（按键去重计数，守「不静默」）。验收：单测——空 support 假设 → `hypothesis_unanchored` 计数且缺席三列表；碰撞歧义键 → added/resolved 正确、不产 confidence_changed、`hypothesis_ambiguous_keys` 计入该键（非 0）。
- [x] 2.3 门继承（D-5）：hypothesis 段仅在未触发任何 `diff_skipped_reason` 早返与跨 target `ValueError` 时计算；任一门触发时三列表与 `hypothesis_unanchored` / `hypothesis_ambiguous_keys` 取零值（**不**扩 `diff_skipped_reason` 闭集、**不**为 hypothesis 单开门）；inspector 版本对齐（rule 5）对 hypothesis 键无过滤（键用假设原样声明的 supporting_findings，**不**重写、**不**校验 id 是否当前可解析）。验收：单测——baseline 非 ok（未 force）/ schema 不一致 / 含 None finding id / 跨 target 时，hypothesis 三列表与 unanchored / ambiguous_keys 全零值或随 `ValueError` 拒绝；现有 finding 级 diff 测试全绿（零回归）。
- [x] 2.4 inspector-skew 可见性（D-5/A）：构造 baseline/current 间有 inspector 升级、且某 hypothesis 引用被 rule 5 排除的 finding —— 断言键不重写、该假设照常参与集合差（v1 行为）；CLI 在 `inspector_upgraded` 非空时输出 caveat 行（见 3.1）。验收：单测断言键不变 + 集合差结果；CLI 测断言 caveat 子串存在。
